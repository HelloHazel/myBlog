## 노드와 네트워크

- 정보를 주고 받는 모든 네트워크에 있는 기기(하드웨어)
- 모든 노드의 한가지 공통점은 바로 **주소**가 있다는 것
- 네트워크가 다른 네트워크를 연결할 때 중간 노드의 역할을 하는 **라우터**
- **라우터**는 IP 패킷을 다른 네트워크로 옮기는 역할

## 서버와 클라이언트

- 서비스를 **제공하는** 노드는 **서버**
- 서버가 제공한 서비스를 **사용하는** 네트워크 노드는 **클라이언트**

## 네트워크의 유형

- 중앙집중식 네트워크
  - 여러 클라이언트가 하나의 라우터에 연결
- 비집중식 네트워크 (분권화된 네트워크)
  - 여러 클라이언트가 여러 라우터에 연결
- 분산 네트워크
  - 모든 노트가 계층 없이 서로 연결 (중앙노드x)

## 하드웨어 주소

- 네트워크 카드
  - 네트워크에 디바이스 ID를 제공하고 **기기와 네트워크 사이의 연결을 처리**함
  - 노드를 인터넷에 연결하려면 반드시 필요함. 왜냐?
  - 네트워크 카드에는 MAC 주소가 있기 때문
- MAC (Media Access Control Adress) 주소
  - 네트워크에 사용되는 모든 기기가 지니고 있고 물리적 식별이 가능한 **고유주소**
  - 기기의 정체성과도 같아서 **디바이스 ID** 라고 부르기도 한다
  - 주로 로컬 네트워크 (LAN)에서만 사용되고 광역 네트워크에서는 잘 사용되지 않는다.
- 임의 MAC 주소
  - 어떤 운영체는 보안 상의 이유로 MAC 주소를 임의로 지정하는 방식을 쓰기도 함

## 기기가 네트워크에 연결되는 방법

- 이더넷 케이블 또는 와이파이로 홈 라우터에 연결하는 방식이 흔하다
  - 이더넷 - 소규모 네트워크를 구성하는 방법 중에 가장 널리 사용되는 유선연결방식
  - 와이파이 - 모바일 기기는 대부분 이더넷 대신 와이파이를 사용
- 라우터에 신호 보내기
  - 다른 노드와 통신하려면 MAC 주소 외에 네트워크 주소가 추가로 필요함
    - 케이블을 이용한 물리 유선 연결하는 방식
    - 무선 연결 방식인 와이파이 네트워크 연결 방식 :
      - 1.  **DHCP**(동적 호스트 구성 프로토콜)을 통해 기기의 네트워크 카드에 따로 네트워크 주소가 할당
      - 2. 요청 성공시 네트워크 주소를 기기에 전송, 로컬 네트워크의 표준 게이트 웨이를 통해 기기에 신호를 보내 네트워크에 전송할 데이터를 요구
      - 3. 표준 게이트웨이를 통해 인터넷에 연결
- 연결하기
  - 네트워크 카드의 MAC에 주소가 할당되면 연결 성공!
  - 이후에는 같은 네트워크에 연결되고 주소가 할당된 다른 노드와 정보를 전송, 전달이 가능해짐

## 패킷이란?

- 패킷
  - 발송정보와 콘텐츠의 두체계로 이루어진 데이터 단위
  - 인터넷 프로토콜은 모든 데이터의 구조를 해체 후, 여러개의 패킷으로 쪼개 전송한다
- 주소 태그
  - 각 패킷에는 발신 및 수신 주소가 포함된 주소 태그가 지정
- 패킷 헤더
  - 패킷의 크기와 유형을 설명하는 역할

## 패킷은 무엇으로 구성될까?

- 패킷은 컴퓨터 언어 -> 이진수 데이터로 구성
- 네트워크 디바이스는 이진수로만 된 정보를 전송, 매체에 따라 다양하게 인코딩함

## 패킷 전송

- 이진수 신호는 주파수 변조라는 과정을 걸쳐 전송
